## [2.0.0] - 2026-04-16 (Release Candidate 最终交付版)

### Added (新增)

- **跨核物理联动**：全面集成 `rpmsg_node.cpp`，打通主核 Linux 与从核裸机的 OpenAMP 通信通道。
- **UI状态防抖**：在 Qt 界面中新增 `hardwareStatusLabel`，实时监控底层蜂鸣器的电平挂起与触发状态。
- **综合状态监控**：引入读取 `/proc/stat` 和 `/sys/class/thermal/thermal_zone0/temp` 的逻辑，实现 CPU 综合利用率和核心温度的每2秒动态刷新预警。

### Fixed (修复)

- 修复了主程序意外退出时，底层蜂鸣器持续长鸣无法停止的致命 Bug。在 `~RPMsgController()` 的析构阶段强制下发了一帧 `data[0] = '0'` 的归零安全指令。
- 修复了因为 `O_NONBLOCK` 非阻塞写入 `/dev/rpmsg0` 导致在高并发场景下警报指令偶发丢包的问题。
- 修复了 Qt 主线程在长时间运行后偶发的内存泄漏问题，通过优化 `QImage` 到 `QPixmap` 的转换逻辑并强制资源释放解决。

### Changed (变更)

- 将所有 AI 模型的加载路径、视频输入路径重构为统一的全局配置管理，彻底消除硬编码带来的跨机器移植崩溃风险。

## [1.8.0] - 2026-04-14 (异构通信底层预研与排雷)

### Added (新增)

- **RPMsg 节点初始化**：尝试通过 `ioctl(RPMSG_CREATE_EPT_IOCTL)` 挂载 `rpmsg-openamp-demo-channel`。
- **通信结构体定义**：确立了 `data_packet` 的底层指令封装格式。

### Fixed (修复)

- **[重大架构修复]** 解决了主核发送指令后从核（Cortex-M）解析为乱码甚至触发 HardFault 的致命问题。通过引入 `#pragma pack(push, 1)` 强制取消 64位 ARMv8 编译器的内存对齐填充（Padding），实现了跨平台 1 字节严格对齐。
- 增加了 RPMsg 连接状态检测，防止在 `/dev/rpmsg_ctrl0` 节点尚未就绪时程序直接崩溃。

## [1.5.0] - 2026-04-12 (核心推理与异步并发架构成型)

### Added (新增)

- **异步推理流水线**：重写 `main.cpp` 调度器，引入 `std::thread` 与 `std::mutex`，将视频拉流（`camera_node`）、推理分析（`inference_node`）剥离至独立核心并行处理。
- **图像零拷贝机制**：在 `ui_main_window.cpp` 中通过 `qRegisterMetaType<cv::Mat>`，实现 OpenCV 矩阵与 Qt GUI 线程的异步队列安全传递。

### Fixed (修复)

- 解决了 Qt UI 线程与推理线程强行抢占 0 号核心算力，导致界面帧率断崖式掉帧（卡顿至 5FPS 以下）的性能瓶颈。
- 修复了多线程环境下 `cv::Mat` 共享内存被污染导致的画面撕裂问题，引入了深度拷贝（`clone()`）与引用计数的平衡机制。

## [1.4.0] - 2026-04-09 (ByteTrack 多目标追踪整合)

### Added (新增)

- **多目标追踪**：完整引入 `ByteTrack` 算法目录，通过卡尔曼滤波（Kalman Filter）与匈牙利算法实现连续视频帧的目标 ID 重识别。

### Fixed (修复)

- 极大减少了同一违规目标在同一区域徘徊时被系统疯狂重复抓拍的问题。
- 调整了卡尔曼滤波的惩罚矩阵系数，缓解了目标被短暂遮挡后 ID 频繁跳变（ID Switch）的缺陷。

## [1.2.0] - 2026-04-04 (数据持久化与工程规范化)

###  新增

- **持久化取证**：新建 `io_node.cpp` 模块，新增违规画面自动截取与带时间戳落盘功能，输出至 `violations_data/` 目录。
- **工程化编译构建**：废弃单文件 GCC 编译，建立全局的 `CMakeLists.txt`，规范化链接 OpenCV、Qt5、NCNN 以及多线程 pthread 库。

### 变更

- 将单帧分析逻辑改为缓冲队列模式，为后续引入的多线程架构做缓冲区预研。

### Known Issues (已知问题)

- 同步 IO 写入磁盘操作会阻塞主线程，导致在产生抓拍图片的那一帧画面会出现肉眼可见的卡顿。

## [1.0.0] - 2026-03-26 (MVP 原型机打通)

### Added (新增)

- **基础 AI 推理**：完成基于 Tencent NCNN 框架的 YOLO 模型量化（INT8）加载与前向计算打通。
- **裸数据流获取**：实现使用 OpenCV 接入 USB 摄像头或本地视频文件的抽帧逻辑。
- **跨平台编译辅助**：编写 `setup_env.sh`。

### Known Issues (已知问题)

- 纯命令行终端输出，缺乏图形化界面，展示效果极差。
- 单核满载运行，CPU 利用率分配极度不均，飞腾派异构多核优势被完全埋没。
